Foundation

What is Reinforcement Learning?

Reinforcement Learning (RL) is a branch of machine learning where an agent learns to make decisions by interacting with an environment. Unlike supervised learning, where a teacher provides correct answers, an RL agent learns from the consequences of its own actions — through trial and error.

The core loop is beautifully simple. At each time step, the agent observes the current state of the environment, takes an action, and receives two things back: a reward signal (a number indicating how good or bad that action was) and the new state it has transitioned into.

The agent's single objective: maximise the total reward it accumulates over time. Not just the immediate reward from the next action, but the cumulative sum of all future rewards. This is what makes RL fundamentally different from greedy optimisation — the agent must learn to sacrifice short-term reward for long-term gain.

Agent

The learner and decision-maker. It observes, acts, and updates its strategy based on rewards received.

Environment

Everything outside the agent. It receives actions and returns new states and reward signals.

Reward Function

A scalar signal from the environment after each action. The agent's goal is to maximise the sum of these over time.

Policy

The agent's strategy — a mapping from states to actions. Learning a good policy is the central challenge of RL.

Value Function

An estimate of how much total future reward the agent can expect from a given state (or state-action pair).

Exploration vs Exploitation

The fundamental tension: should the agent try something new (explore) or stick with what it knows works (exploit)?

This framework turns out to be extraordinarily general. RL is how DeepMind's AlphaGo learned to defeat the world champion at Go — a game with more board positions than atoms in the universe. It is how self-driving systems learn to navigate roads they have never seen. It powers YouTube's recommendation algorithm, deciding which video to show you next to maximise engagement. In finance, RL agents learn exploratory pricing strategies and dynamic portfolio allocation. In fraud detection, RL decides which suspicious cases to escalate so the system catches novel fraud patterns rather than only the ones it already knows about. Wherever there is a sequence of decisions under uncertainty, RL is a natural fit.

A word of honesty: most RL success stories depend heavily on simulation. AlphaGo played millions of games against itself. Self-driving stacks train in virtual cities before touching real roads. Off-policy methods like Q-learning are powerful precisely because they can learn from simulated or historical data without ever deploying a live policy. On-policy learning in the real world — where the environment drifts, data is expensive, and mistakes have consequences — is a genuinely hard problem and an active area of research. The non-stationary bandits section below gives you a small taste of why.

In the sections that follow, we'll build intuition for each of these concepts interactively. We start with the multi-armed bandit — the simplest RL setting, where there are no states, just a single repeated decision. Then we introduce Markov Decision Processes, which add states and transitions, and explore how agents learn optimal behaviour through Q-learning.

Part I

The Multi-Armed Bandit Problem

Imagine you're standing in front of a row of slot machines. Each machine pays out at a different, unknown rate. You have a limited number of pulls. How do you maximise your total reward?

This is the multi-armed bandit (MAB) problem — the simplest meaningful setting in reinforcement learning. There are no states, no transitions, just a single repeated decision: which arm do I pull next?

Each arm k has a true mean reward q*(k) which is unknown to the agent. The agent maintains an estimate Q(k) based on observed rewards. Below you can see a randomly generated set of arm values — the hidden truth the agent must discover. The whiskers show the ±1σ reward distribution around each arm's mean.

Key Insight: Every exploration strategy represents a different answer to: "How should I balance what I know against what I might learn?"

We compare four classic strategies:

Pure Greedy always picks the arm with the highest current estimate. ε-Greedy picks the best arm most of the time but explores randomly with probability ε. UCB selects the arm that maximises estimated value plus an exploration bonus. Boltzmann converts Q-values into a probability distribution using temperature τ.

ε-Greedy:   A = argmax_a Q(a) with prob (1-ε),   random with prob ε

UCB1:      A = argmax_a [ Q(a) + c × √( ln(t) / N(a) ) ]

Boltzmann: P(a) = exp(Q(a)/τ) / Σ_b exp(Q(b)/τ)

Simulation: Compare Exploration Strategies

The environment: a row of k slot machines. Each arm has a hidden true payout drawn from a normal distribution N(0, 1). When you pull arm a, you receive a reward drawn from N(q*(a), 1) — the true value plus noise. The agent cannot see the true values; it must estimate them from experience. The goal is to accumulate the most reward over a fixed number of steps.

Below are the true arm values q*(a) with ±1σ reward distributions. This is what the agent must discover through interaction.

Arms — the number of slot machines (actions). More arms means a harder exploration problem because the agent has more options to try before it can be confident it has found the best one. Steps — how many total pulls the agent gets. Short horizons punish over-exploration; long horizons let even slow strategies converge. Runs — independent repetitions averaged together. Averaging smooths out randomness so you see the true performance differences between strategies rather than noise.

ε (epsilon) — the probability of exploring a random arm instead of exploiting the current best estimate. Higher ε means more exploration, which helps find the best arm faster but wastes more pulls on bad arms. Too low and the agent may never discover a superior arm. UCB c — the exploration weight in Upper Confidence Bound. Larger c gives more bonus to under-explored arms, encouraging the agent to try uncertain options before committing. Boltzmann τ (temperature) — controls how sharply the agent favours high-value arms. High τ makes the selection nearly uniform (maximum exploration). Low τ approaches pure greedy.

Part II

Non-Stationary Environments

So far we assumed each arm has a fixed true reward. But what if the environment changes? In a non-stationary problem, the true values drift over time — an arm that was great yesterday might be mediocre today.

The key implication: agents must never stop exploring. Strategies that gradually reduce exploration converge to stale policies. Strategies that maintain constant exploration, or that weight recent observations more heavily, adapt far better.

Key Insight: In non-stationary environments, forgetting old data is just as important as collecting new data. A constant step-size α (exponential recency-weighted average) naturally down-weights stale observations.
Sample average (stationary):   Q(a) ← Q(a) + (1/N(a)) × [R - Q(a)]

Constant step-size (non-stationary):   Q(a) ← Q(a) + α × [R - Q(a)]

Simulation: Tracking a Drifting Target

The environment: identical to the stationary bandit above, except that after every single step each arm's true value drifts by a small random amount drawn from N(0, σ). Over time these random walks cause the best arm to change unpredictably. An agent that stopped exploring after "solving" the initial problem will keep pulling an arm whose value has drifted away from the top.

Drift σ — the standard deviation of the random walk applied to each arm per step. Larger σ means the environment changes faster, making the problem harder and penalising agents that rely on stale estimates. Step-size α — the constant learning rate used by the adaptive strategy. Instead of averaging all past observations equally (sample average), a constant α gives exponentially more weight to recent rewards. This is what lets the agent track a moving target — old observations are naturally forgotten as new ones arrive.

Part III

Markov Decision Processes

Bandits are a single-state problem. But most interesting RL problems have sequences of decisions where each action changes the world. The framework for this is the Markov Decision Process (MDP).

An MDP is defined by (S, A, P, R, γ):

  S = set of states    A = set of actions
  P(s'|s,a) = transition probabilities    R(s,a,s') = rewards
  γ = discount factor (0 ≤ γ ≤ 1)

The Markov property is key: the next state depends only on the current state and action, not on history. The agent's goal is to learn a policy π(s) → a that maximises the expected discounted return:

G_t = R_{t+1} + γ R_{t+2} + γ² R_{t+3} + ...

γ → 0: myopic (immediate reward only)
γ → 1: far-sighted (values future rewards equally)

The Bellman equation connects a state's value to its successors recursively. It's the foundation of Q-learning:

Q*(s, a) = E[ R + γ × max_a' Q*(s', a') ]

Q-learning update:
  Q(s, a) ← Q(s, a) + α × [ R + γ × max_a' Q(s', a') − Q(s, a) ]

That update rule is an instance of Temporal Difference (TD) learning — the idea that you can learn from incomplete experience. Instead of waiting until an episode ends to find out the total reward (as in Monte Carlo methods), TD updates the value estimate after every single step, using the difference between what you expected and what you actually observed plus your estimate of what comes next. The quantity R + γ Q(s', a') − Q(s, a) is called the TD error — it is the surprise signal that drives learning. Both Q-learning and SARSA (which you will see in the Ice Walk section) are TD methods; they differ only in which next action they use to form the target.

Key Insight: A grid world is a perfect MDP. Each cell is a state, directions are actions, transitions are deterministic, and the Markov property holds because where you should go next depends only on where you are, not how you got there.
Part IV

Grid World & Q-Learning

Now we apply Q-learning to an MDP. The agent navigates a grid to reach the goal (★). Watch Q-values propagate backward from the goal — cells near it light up first, then value information flows to every reachable state via the Bellman updates.

What you are watching is the credit assignment problem in action — arguably the central difficulty of RL. When an agent reaches the goal after 40 steps, which of those steps actually mattered? The Bellman backup solves this by propagating reward information backward through the state space, one step at a time. Cells adjacent to the goal learn their value first. Then cells adjacent to those cells inherit it, and so on, until every reachable state knows roughly how far it is from reward. This is slow, and it is why the early episodes look terrible — the agent is wandering in the dark until value has had time to propagate back to where it starts.

The environment: an N×N grid with the agent starting at the top-left corner and the goal at the bottom-right. Some interior cells are randomly blocked as walls (~15% of the grid). The agent can move up, right, down, or left; moving into a wall or off the edge does nothing. Reaching the goal gives +10 reward. Every other step costs −0.1, which encourages the agent to find the shortest path rather than wander aimlessly.

Grid Size — the dimensions of the world. Larger grids have exponentially more states, which means the agent needs more episodes to propagate value information from the goal all the way back to the start. Episodes — how many complete start-to-goal (or timeout) attempts the agent gets. Each episode resets the agent to the start but preserves learned Q-values.

Learning rate α — how much each new experience overwrites the old Q-value estimate. High α learns fast but is noisy. Low α is stable but slow to adapt. In a deterministic grid like this, higher values (0.1–0.5) work well. Discount γ — how much the agent values future reward relative to immediate reward. At γ=0.95, a reward 20 steps away is worth 0.95^20 ≈ 36% of its face value. Lower γ makes the agent myopic; higher γ makes it plan further ahead. Exploration ε — the probability of taking a random action instead of the greedy best. Without exploration the agent may never discover shorter paths. Too much exploration wastes episodes on known-bad moves.

Part V

The Ice Walk: SARSA vs Q-Learning

The Ice Walk is a classic RL benchmark that illustrates the difference between on-policy (SARSA) and off-policy (Q-learning) methods. The agent must cross a frozen ridge from Start (S) to Goal (G) along the bottom of a 4×12 grid. Between them runs a stretch of thin ice — step on it and the ice cracks, sending the agent plummeting back to the start with a brutal −100 penalty. Every safe step costs −1.

Q-learning discovers the shortest path — right along the ice edge — because that is objectively the fastest route. But during training, random exploratory steps sometimes send the agent onto the thin ice, accumulating massive penalties. SARSA learns a safer path along the top of the grid because it evaluates the policy it is actually following, including the exploratory mistakes it knows it will make. It "prices in" its own clumsiness.

Q-learning:   Q(s,a) ← Q(s,a) + α[R + γ max_a' Q(s',a') − Q(s,a)]

SARSA:       Q(s,a) ← Q(s,a) + α[R + γ Q(s',a') − Q(s,a)]

Difference: SARSA uses the actual next action a', Q-learning uses the best.
Key Insight: Q-learning says "I know the best path even if I stumble during training." SARSA says "I'd rather take a safe path because I know I'll make mistakes."

Simulation: On-Policy vs Off-Policy

The environment: a 4×12 grid. The agent starts at the bottom-left cell. The goal is the bottom-right cell. The entire bottom row between them (10 cells) is thin ice — stepping on any of those cells gives −100 reward and teleports the agent back to the start. All other transitions cost −1. The episode ends when the agent reaches the goal.

Episodes — the number of complete attempts. Both SARSA and Q-learning train on the same number of episodes so the comparison is fair. Learning rate α — set higher here (default 0.5) than in the grid world because the cliff walking environment is small and deterministic, so aggressive updates converge faster. Discount γ — set to 1.0 by default because there is no reason to discount future steps in this task; every step has equal weight and the episodes are short. Exploration ε — this is the parameter that creates the interesting difference between SARSA and Q-learning. SARSA knows it will take random actions with probability ε, so it avoids states where a random step would be catastrophic. Q-learning ignores the exploration policy when evaluating states, so it values the ice-edge path at its theoretical optimum.

What We Left Out

Where Does RL Go From Here?

Everything in this tutorial is tabular — every state gets its own row in a table of Q-values. That works for a 7×7 grid (49 states) or a 4×12 cliff (48 states). It does not work for chess (roughly 1047 states), Go (10170), a camera image (millions of pixels), or a self-driving car's sensor feed. The state space is simply too large to visit every entry even once, let alone update it thousands of times.

The solution is function approximation — replacing the Q-table with a function that generalises across similar states. A neural network, for instance, can take a raw image as input and output Q-values for every action, sharing learned features across states it has never seen before. This is where deep learning enters: Deep Q-Networks (DQN) combine the Q-learning update you saw above with a convolutional neural network. It is what let an agent learn to play Atari games from raw pixels in 2015.

Beyond value-based methods, policy gradient methods learn the policy directly — adjusting the probability of actions based on how much reward they lead to, without ever maintaining a Q-table. Algorithms like PPO and TRPO belong here, and they are the workhorses behind RLHF (reinforcement learning from human feedback), which is how large language models are fine-tuned to be more helpful and less harmful.

There is also model-based RL, where the agent learns a model of the environment itself — predicting what will happen next — and uses that model to plan ahead rather than learning purely from trial and error. AlphaZero is a model-based system: it knows the rules of the game and searches forward through possible futures.

None of these methods are covered here, and that is deliberate. They are built on top of the ideas you just worked through — value estimation, the Bellman equation, TD learning, exploration, on-policy versus off-policy. If those ideas are solid, everything else is a variation on theme. If they are not, no amount of neural network architecture will save you.
